# 批处理文件使用说明

## 问题说明

之前的批处理文件（.bat）包含中文字符，在某些 Windows 系统上会出现乱码，导致命令无法正确执行。

## 已修复

现在所有批处理文件已更新为：
- ✅ 使用 `chcp 65001` 设置 UTF-8 编码
- ✅ 所有提示信息改为英文
- ✅ 优化了错误处理和依赖检查

## 使用方法

### 后端设置和启动

1. **首次设置**（只需运行一次）：
   ```cmd
   cd backend
   setup.bat
   ```
   这将：
   - 创建 Python 虚拟环境
   - 安装所有依赖包
   - 创建 .env 配置文件

2. **启动开发服务器**：
   ```cmd
   cd backend
   start.bat
   ```
   服务器将在 http://localhost:8000 启动

### 前端启动

```cmd
cd frontend
start.bat
```
前端将在 http://localhost:5173 启动

## 如果仍然出现问题

### 方法 1: 手动运行命令

**后端：**
```cmd
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**前端：**
```cmd
cd frontend
npm install
npm run dev
```

### 方法 2: 使用 PowerShell

如果批处理文件仍有问题，可以使用 PowerShell：

**后端：**
```powershell
cd backend
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**前端：**
```powershell
cd frontend
npm install
npm run dev
```

## 常见问题

### Q: 提示 "Python not found"
A: 请先安装 Python 3.11 或更高版本，并确保添加到系统 PATH

### Q: 提示 "Node.js not found"
A: 请先安装 Node.js 18 或更高版本

### Q: pip install 很慢
A: 可以使用国内镜像源：
```cmd
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q: npm install 很慢
A: 可以使用 cnpm 或配置淘宝镜像：
```cmd
npm config set registry https://registry.npmmirror.com
npm install
```
