# 📄 拼豆豆助手 - 首页重构设计规范 (V2.0)

**文档状态**：定稿 / 待设计执行
**设计核心哲学**：“UI 是隐形的，用户的拼豆作品才是唯一的舞台。”

---
## 1. 设计目标

首页不是功能目录，也不是运营广告位。

首页的核心目标只有一个：

> 让用户在最短时间内进入下一步创作。

### 原则 1：首页必须有唯一主动作
打开首页后，用户应在 3 秒内知道自己下一步做什么。

### 原则 2：系统替用户做判断
根据用户状态动态调整首页核心区域，而不是让所有入口永远平权。

### 原则 3：任务优先，不展示功能
入口文案要从“功能命名”转向“任务命名”。

### 原则 4：作品是主角，不是 UI 是主角
界面配色和组件装饰应克制，让作品缩略图和创作状态成为视觉焦点。

### 原则 5：信息必须能转化为行动
每个模块都必须具备明确下一步，而不是只是展示数据。
## 一、 全局视觉与交互原则 (Global Rules)

在画具体的像素之前，请团队所有人记住以下 3 条绝对原则：

1. **色彩克制 (Color Discipline)**
   - **全面封杀 UI 渐变色**（特别是旧版中廉价的紫红色 Banner）。
   - App 的背景色必须是极具高级感的**苹果灰 (`#F5F5F7`)** 或 **纯白 (`#FFFFFF`)**。
   - 唯一的亮色应该来自**用户的拼豆图案本身**。UI 按钮的主色调统一为一种有质感的单色（如纯黑 `#1C1C1E` 或 活力品牌色）。
2. **物理质感 (Physical Tactility)**
   - 拼豆是手工艺术。UI 中需要加入隐喻：比如极浅的**底板圆点暗纹**、作品缩略图的**微弱立体投影 (Drop Shadow)**。
   - 让用户感觉隔着屏幕能“摸”到豆子，而不是在用一个文件管理器。
3. **消除焦虑 (Zero Anxiety)**
   - 首页不展示“缺货、库存报警、数据统计”。首页只做一件事：**激发创作欲望**。

## 二、 页面结构拆解 (Section by Section)

从上到下，首页仅保留四个核心模块。

### 1. 极简导航栏 (Header)
- **左侧**：精简版 Logo + 产品名（深灰色，不要五颜六色）。
- **右侧**：搜索 Icon 🔍 + 消息 Icon 🔔（采用轻量的线性图标，有未读消息时给一个极小的红点即可）。

### 2. 核心唤醒区：魔法底板 (The Magic Canvas)
*（替代旧版的紫色 Banner + 4个呆板的工具箱图标）*

这是一个占据屏幕上方约 1/3 面积的巨大交互卡片，它长得像一块真实的”空白拼豆底板”。

#### 2.1 视觉设计

- **背景**：带有一点点微凹效果的透明圆点网格（隐喻拼豆底板）。
- **主视觉**：正中央放一个极简的「相机/相册」融合图标。
- **主文案**：`+ 唤醒灵感`
- **副文案**：`拍照 或 从相册导入图片`
- **辅助文案**：`AI 魔法，一键生成专属图纸`（小字，灰色）
- **新手入口的归宿**：在卡片右上角或底部，放一个非常轻量的文字链 `新手？点击体验 10 分钟入门 >`，不要用巨大的按钮去打扰老用户。

#### 2.2 交互设计：扁平化动作表单

**设计原则：** 减少决策层级，让用户一步到位选择创作来源。

点击”唤醒灵感”卡片的任何区域 → 弹出底部动作表单（Action Sheet）

**动作表单结构：**

```
┌─────────────────────────────┐
│  开始创作                    │
├─────────────────────────────┤
│  📷  拍照转图纸              │
│  🖼️  从相册选择              │
│  📁  我的图纸库 (3张) →      │
│  🎨  从模板开始 →            │
├─────────────────────────────┤
│  取消                        │
└─────────────────────────────┘
```

**选项说明：**

| 选项 | 图标 | 说明 | 跳转目标 |
|------|------|------|----------|
| 拍照转图纸 | 📷 | 调用系统相机拍照 | → 创作Tab（图片裁剪 → 转图流程） |
| 从相册选择 | 🖼️ | 打开系统相册选择图片 | → 创作Tab（图片裁剪 → 转图流程） |
| 我的图纸库 | 📁 | 显示已保存的图纸数量，点击进入图纸选择器 | → 图纸选择器 → 编辑器 |
| 从模板开始 | 🎨 | 弹出精选模板弹窗（6-9个模板） | → 模板精选弹窗 → 社区Tab |
| 取消 | - | 关闭动作表单，返回首页 | - |

#### 2.3 “我的图纸库”状态逻辑

**状态1：无图纸**
- 显示：`📁 我的图纸库` （灰色文字）
- 副标题：`暂无图纸`
- 点击后：提示”还没有图纸，先转换一张图片吧”

**状态2：有图纸（1-5张）**
- 显示：`📁 我的图纸库 (3张) →`
- 点击后：弹出图纸选择器（横向滑动卡片）

**状态3：图纸库已满（免费版≥5张）**
- 显示：`📁 我的图纸库 (5张/已满) →`
- 点击后：提示”图纸库已满，请删除旧图纸或升级专业版”

#### 2.4 智能提示（可选功能）

**场景1：有未完成作品时**
- 点击”唤醒灵感” → 先弹出提示弹窗
- 文案：`你有 1 个未完成的作品，要继续吗？`
- 按钮：`[继续制作]` `[开始新作品]`
- 点击”开始新作品” → 显示动作表单

**场景2：图纸库有新内容时**
- “唤醒灵感”卡片右上角显示小红点
- 动作表单中”我的图纸库”选项高亮显示

**场景3：经常使用模板的用户**
- 动作表单中”从模板开始”选项置顶
- 显示标签：`上次用过`

#### 2.5 “从模板开始”交互细节

点击”从模板开始” → 不直接跳转到社区Tab，而是弹出模态页（Modal）

**模态页结构：**

```
┌─────────────────────────────┐
│ [✕]        精选模板          │
├─────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐       │
│  │模板│ │模板│ │模板│       │
│  │ 1  │ │ 2  │ │ 3  │       │
│  └────┘ └────┘ └────┘       │
│  ┌────┐ ┌────┐ ┌────┐       │
│  │模板│ │模板│ │模板│       │
│  │ 4  │ │ 5  │ │ 6  │       │
│  └────┘ └────┘ └────┘       │
│                             │
│  [查看更多模板 →]            │
└─────────────────────────────┘
```

- 展示 6-9 个精选模板（新手友好、热门）
- 点击模板 → 模板详情 → 一键开始
- 点击”查看更多模板” → 跳转到社区Tab的模板广场

**优势：**
- ✅ 保持首页沉浸感，不打断用户流程
- ✅ 快速预览，降低决策成本
- ✅ 减少Tab跳转，提升体验流畅度

### 3. 继续上次创作 (In Progress)
*（优化旧版的马里奥蘑菇卡片）*

不要让它看起来像个下载进度条。要让它像**“书桌上未完成的手工作品”**。

- **布局**：横向滑动卡片（如果只有一个，则居中展示）。
- **缩略图**：**必须是带有圆孔和光影质感的拼豆渲染图**，而不是一张扁平的矢量画。背景透明。
- **信息展示**：
  - 标题：`超级马里奥蘑菇` (H2字体大小，加粗)
  - 进度：一条极细的、优雅的进度条（如 65%）。
  - 辅助文字：`距完成约 15 分钟`（比“上次编辑 2 小时前”更有激励感）。
- **操作**：去掉那个显眼的紫色按钮。将整个卡片做成可点击，右侧加一个极简的 `继续 ➔` 箭头即可。

### 4. 末灵感画廊 (Inspiration Gallery)
*（替代旧版的“今日推荐”带框卡片）*

不要用电商商品列表的排版方式来展示艺术品。

- **布局**：瀑布流 (Masonry Grid) 或 横向无缝滑动 (Horizontal Scroll)。
- **视觉**：**去掉白色的卡片底色和生硬的边框**。让拼豆图案直接“悬浮”在浅灰色的背景上，底部带上柔和的阴影。
- **文案情感化**：
  - ❌ 旧版：`简单 8色` / `中等 12色` （太像说明书了）
  - ✅ 新版：`15分钟搞定` / `适合送朋友` / `桌面小摆件` （传递情绪价值）

## 五、"我的图纸库"功能设计

### 5.1 功能定位

**核心价值：** 为用户提供已转换图纸的存储和管理功能，支持快速复用和二次创作。

**与"我的作品"的区别：**
- **我的图纸** = 已转换的图纸文件（未开始制作，纯数据）
- **我的作品** = 正在制作或已完成的作品（有进度状态、跟做记录）

**流程关系：**
```
原始图片 → [转换] → 图纸 → [开始制作] → 作品
```

### 5.2 图纸选择器页面

从动作表单点击"我的图纸库" → 弹出图纸选择器（半屏模态页）

**页面结构：**

```
┌─────────────────────────────┐
│ [< 返回]  我的图纸库  [+ 上传]│
├─────────────────────────────┤
│  ┌────────┐ ┌────────┐      │
│  │ [图纸] │ │ [图纸] │ → → │
│  │   1    │ │   2    │      │
│  │ 29×29  │ │ 15×15  │      │
│  │ 14色   │ │  8色   │      │
│  │ 2天前  │ │ 5天前  │      │
│  └────────┘ └────────┘      │
│                             │
│  点击图纸直接进入编辑器       │
│  长按可删除或重命名           │
└─────────────────────────────┘
```

**页面元素：**

| 元素 | 说明 |
|------|------|
| 返回按钮 | 返回首页 |
| + 上传按钮 | 上传已有图纸（PNG/PDF/图片） |
| 图纸卡片 | 横向滑动列表，显示图纸缩略图 |
| 图纸信息 | 尺寸、颜色数、创建时间 |
| 操作提示 | 底部灰色小字提示 |

**图纸卡片交互：**
- **点击** → 直接进入编辑器（可编辑后保存为作品）
- **长按** → 弹出操作菜单：
  - 重命名
  - 删除
  - 分享到社区（需审核）
  - 导出图纸

### 5.3 图纸上传功能

点击"+ 上传"按钮 → 弹出上传选项

**上传选项：**

```
┌─────────────────────────────┐
│  上传图纸                    │
├─────────────────────────────┤
│  📄  上传图纸文件 (PNG/PDF)  │
│  🖼️  上传图片并转换          │
├─────────────────────────────┤
│  取消                        │
└─────────────────────────────┘
```

**选项1：上传图纸文件**
- 支持格式：PNG（带色号标注）、PDF（拼豆图纸导出格式）
- 自动识别：如果是已转换的图纸，直接导入
- 解析失败：提示"无法识别图纸格式，请选择标准图纸文件"

**选项2：上传图片并转换**
- 选择普通图片 → 跳转到转图流程
- 转换完成后自动保存到图纸库

**批量上传（可选功能）：**
- 支持一次选择多张图片（最多5张）
- 后台批量转换，显示进度
- 转换完成后通知用户

### 5.5 容量限制与付费策略

**免费版限制：**
- 图纸库容量：最多 5 张
- 单张图纸大小：≤ 58×58
- 云端存储：7 天后自动删除

**专业版权益：**
- 图纸库容量：无限制
- 单张图纸大小：无限制
- 云端存储：永久保存
- 批量上传：支持

**触发付费引导：**
- 图纸库已满时，弹窗提示：
  ```
  图纸库已满 (5/5)
  升级专业版可保存无限图纸

  [升级专业版]  [管理图纸]
  ```

### 5.6 图纸分享功能（可选）

**分享到社区：**
- 用户可将自己转换的图纸分享到社区模板广场
- 需要审核（防止版权问题）
- 分享时自动添加水印（免费版）
- 分享后可获得积分奖励

**分享限制：**
- 从模板保存的图纸：不可分享（避免重复）
- 上传的图纸：需声明版权归属
- 转换的图纸：可分享，但需遵守社区规范

### 5.7 图纸版本管理（可选功能）

**场景：** 用户在编辑器中修改图纸后，是否覆盖原图纸？

**方案A：覆盖原图纸**
- 优点：简单，不占用额外空间
- 缺点：无法回退到原始版本

**方案B：保存为新版本**
- 优点：保留历史版本，可回退
- 缺点：占用更多空间，管理复杂

**推荐方案：** 方案A（MVP阶段）
- 编辑图纸时，弹出提示：
  ```
  保存修改后的图纸？
  [覆盖原图纸]  [另存为新图纸]
  ```
- 默认选择"覆盖原图纸"
- 专业版用户可选择"另存为新图纸"

### 5.8 信息架构更新

**首页 → 唤醒灵感 → 动作表单：**
```
📷 拍照转图纸 → 创作Tab（转图流程）→ 保存到图纸库
🖼️ 从相册选择 → 创作Tab（转图流程）→ 保存到图纸库
📁 我的图纸库 → 图纸选择器 → 编辑器 → 保存为作品
🎨 从模板开始 → 模板精选 → 保存到图纸库
```

**我的Tab → 我的图纸库：**
```
我的Tab
  ├─ 我的作品（正在制作/已完成）
  ├─ 我的图纸库（已转换图纸）← 也可以从这里访问
  ├─ 我的收藏（收藏的模板）
  └─ 设置
```

### 5.9 关键产品决策

**Q1: 图纸和作品的转换关系？**

A: 单向转换，不可逆
- 图纸 → 作品：点击"开始制作"
- 作品 ✗ 图纸：已开始的作品不能退回图纸状态
- 原因：作品包含跟做进度、实拍照片等额外数据

**Q2: 转图后是否自动保存到图纸库？**

A: 是的，但需要用户确认
- 转图完成后，方案选择页增加选项：
  - `[使用这个方案]` → 直接开始制作（保存为作品）
  - `[保存到图纸库]` → 仅保存图纸，稍后使用
- 默认行为：点击"使用这个方案"时，同时保存到图纸库

**Q3: 图纸可以编辑吗？**

A: 可以，但编辑后需要选择保存方式
- 点击图纸 → 进入编辑器（只读模式）
- 修改后 → 弹出保存选项：
  - 覆盖原图纸
  - 另存为新图纸
  - 保存为作品（开始制作）

## 六、交互流程图

### 6.1 唤醒灵感完整流程

```
首页
  ↓
点击"唤醒灵感"
  ↓
┌─────────────────────────────┐
│  动作表单                    │
│  ├─ 📷 拍照转图纸            │
│  ├─ 🖼️ 从相册选择            │
│  ├─ 📁 我的图纸库 (3张)      │
│  └─ 🎨 从模板开始            │
└─────────────────────────────┘
  ↓
分支1: 拍照/相册
  → 创作Tab（图片裁剪）
  → 方案生成
  → 方案选择
  → [使用方案] or [保存到图纸库]

分支2: 我的图纸库
  → 图纸选择器
  → 选择图纸
  → 编辑器
  → [保存为作品] or [覆盖图纸]

分支3: 从模板开始
  → 模板精选弹窗
  → 选择模板
  → 模板详情
  → [一键开始] → 保存为作品
```

### 6.2 图纸生命周期

```
[原始图片]
    ↓ 转换
[图纸] ← 可编辑、可删除、可分享
    ↓ 开始制作
[作品] ← 有进度、有跟做记录
```
