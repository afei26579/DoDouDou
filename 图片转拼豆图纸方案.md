# 图片转拼豆图纸方案（专家版）

> 目标：在“视觉还原”和“制作可行”之间取得最优平衡，输出可直接用于拼豆制作的高质量图纸。

---

## 1. 设计原则

1. **感知优先**：颜色匹配以人眼感知一致性为核心（Lab + CIEDE2000）。
2. **制作优先**：减少碎点、孤岛和过多颜色，提升可拼性。
3. **参数可控**：提供少量高价值参数，保证新手可用、高手可调。
4. **性能可预期**：大图不阻塞，支持分块与异步计算。

---

## 2. 总体流水线

```text
输入图片
  → 预处理（降噪/主体裁剪/颜色空间转换）
  → 网格采样（自适应：均值+主导色）
  → 调色板量化（映射到拼豆色库）
  → 制作优化（去孤岛/小连通域合并/边缘保护）
  → 输出图纸（店家色号+统计+拼制引导）
```

---

## 3. 核心算法设计

### 3.1 输入预处理

- **尺寸标准化**：按目标难度设置网格宽度（如 72/96/120）。
- **边缘保留降噪**：优先 bilateral 或小核中值滤波，抑制噪点但保轮廓。
- **颜色空间转换**：统一到 Lab，作为后续距离计算基础。

**建议参数**
- `targetWidth`: `72 | 96 | 120`
- `denoise`: `true`
- `medianKernel`: `3`

---

### 3.2 网格采样（像素化）

采用**自适应块策略**：

- 低纹理块：使用均值色（平滑、干净）
- 高纹理/边缘块：使用主导色（轮廓更清晰）

可选三种模式：
- `cartoon`：主导色偏重
- `realistic`：均值色偏重
- `adaptive`：自动切换（推荐默认）

**建议参数**
- `edgeRatioThreshold = 0.15`
- `mode = adaptive`

---

### 3.3 调色板映射（关键）

1. 使用统一标准色库（291 色）作为算法层基准。
2. 对标准色预计算 Lab。 
3. 每个采样色用 CIEDE2000 找最近标准色。 
4. 展示层再映射到店家色号（MARD/COCO/漫漫/盼盼/咪小窝）。

**两层映射**
- 算法层：`sampleColor -> standardHex`
- 展示层：`standardHex -> vendorCode`

---

### 3.4 制作可行性优化（拼豆特化）

#### A. 孤立点清理
- 若某像素在 8 邻域中同色支持度过低，则替换为邻域主色。

#### B. 小连通域合并
- 对每个颜色做连通域分析。
- 面积小于阈值（如 `<3` 或 `<6`）的区域，合并到“相邻且色差最小”的大区域。

#### C. 边缘保护
- 对高对比边界区域施加保护，避免过度合并导致轮廓损失。

#### D. 颜色数控制
- 支持“最多 N 色”（建议 12/18/24）。
- 超限时迭代合并视觉差最小且碎片化高的颜色。

---

## 4. 抖动策略

- 默认关闭误差扩散抖动（避免难拼的噪点纹理）。
- 仅在 `realistic` 模式对局部渐变区域启用低强度抖动。
- 启用抖动时，仍需执行最小连通域约束防止单点噪纹。

---

## 5. 输出数据结构（建议）

```ts
interface PatternPixel {
  x: number;
  y: number;
  standardHex: string;
  vendorCode: string;
}

interface PatternResult {
  width: number;
  height: number;
  pixels: PatternPixel[];
  colorStats: Array<{ standardHex: string; vendorCode: string; count: number }>;
  difficulty: {
    colorCount: number;
    tinyRegionCount: number;
    totalBeads: number;
    score: number;
  };
}
```

---

## 6. 参数预设（可直接上线）

### 新手（制作优先）
- `targetWidth = 72`
- `maxColors = 12`
- 强去孤岛 + 强连通域合并

### 标准（默认）
- `targetWidth = 96`
- `maxColors = 18`
- 平衡策略（推荐）

### 进阶（细节优先）
- `targetWidth = 120`
- `maxColors = 24`
- 弱合并 + 可选轻抖动

---

## 7. 质量评估指标

1. **视觉误差**：平均 DeltaE（越低越好）
2. **颜色复杂度**：最终颜色种类数
3. **碎片度**：小连通域数量与占比
4. **制作难度指数**：综合颜色数、碎片度、总颗粒数

可将以上指标组合成总分，用于自动推荐参数。

---

## 8. 性能实现建议

1. **Web Worker**：像素化、映射、连通域分析放入 worker。
2. **分块处理**：大图按 tile 计算，降低峰值内存。
3. **缓存**：
   - 调色板 Lab 缓存
   - 常用颜色距离查表
4. **渐进式反馈**：先出预览，再精修优化。

---

## 9. 迭代落地顺序

### Phase 1（高收益低风险）
- Lab + CIEDE2000 映射替换 RGB 距离
- 调色板 Lab 预计算

### Phase 2
- 孤立点清理 + 小连通域合并
- 边缘保护策略

### Phase 3
- 颜色数约束的全局优化
- 可选低强度抖动

### Phase 4
- Worker + 分块并行 + 性能监控

---

## 11. 结论

这套方案本质是把“图片转图纸”从**单纯最近色匹配**升级为**感知量化 + 可制作性优化**。在不牺牲易用性的前提下，可明显降低手工返修次数，并提升拼豆过程的顺畅度与成品稳定性。
