### 0.5 术语定义

| 术语 | 含义 |
|------|------|
| 作品 | 用户通过导图或模板创建的一个拼豆项目 |

## 1. 原型设计原则

### 1.1 核心目标

> 让用户尽快开始，并尽可能顺利地完成一件拼豆作品。

### 1.2 四条设计原则

| 编号 | 原则 | 具体要求 | 冲突时的优先级 |
|------|------|----------|---------------|
| A | 优先走通主流程 | 导图路径、模板路径、跟做路径必须清晰无断点 | 最高 |
| B | 页面信息简洁 | 每页只承载当前任务最需要的信息，不提前暴露复杂能力 | 高 |
| C | 优先结果预览 | 用户导图后先看到结果，而非参数面板 | 高 |
| D | 跟做页低认知负担 | 跟做页面避免复杂操作，突出当前步骤和进度，适合"做手工时快速扫一眼" | 最高（该页面内） |

## 2. 全局信息架构

### 2.1 一级导航结构

**底部 Tab Bar — 4 个 Tab：**

```
┌──────────┬──────────┬──────────┬──────────┐
│   首页   │  模板库  │ 我的作品 │   我的   │
│   🏠     │   📋    │   🎨    │   👤    │
└──────────┴──────────┴──────────┴──────────┘
```

**导航规则：**
- Tab Bar 在所有一级页面常驻显示
- 跟做模式、编辑器为沉浸式全屏页面，**隐藏 Tab Bar**
- 新建作品通过首页主按钮进入，不占 Tab 位
- 当前 Tab 高亮，未读/进行中状态不在 MVP 显示角标

### 2.2 页面层级关系

```
Layer 0 (Tab)     Layer 1            Layer 2           Layer 3
─────────────────────────────────────────────────────────────

首页 ─────────── 新建作品 ──────── 图片裁剪 ────── 方案生成中
               │                                      │
               │                                  方案选择 ──── 编辑器
               │                                      │
               │                                  作品详情 ──── 跟做模式 ──── 完工页
               │                                      │
               │                                  材料清单(弹层)
               │
               └── 继续未完成作品 → 作品详情 → 跟做模式

模板库 ──────── 模板详情 ──────── 作品详情 → 跟做模式 → 完工页

我的作品 ────── 作品详情 ──────── 跟做模式 → 完工页

我的 ─────────── 我的收藏
               ├── 使用偏好
               ├── 会员中心
               ├── 帮助与反馈
               └── 关于
```

### 2.3 三条主链路

**链路 A：导图创作（核心链路）**

```
首页 → 新建作品 → 导入图片 → 裁剪 → 方案生成中 → 方案选择
  → [可选] 编辑器 → 保存作品 → 作品详情 → 跟做模式 → 完工页
```

**链路 B：模板创作**

```
首页/模板库 → 模板详情 → 一键开始 → 保存为我的作品
  → 作品详情 → 跟做模式 → 完工页
```

**链路 C：继续创作**

```
首页(继续卡片) / 我的作品 → 作品详情 → 继续跟做 → 恢复上次步骤 → 完工页
```

| 操作 | 行为 | 说明 |
|------|------|------|

**页面状态：**

| 状态 | 表现 |
|------|------|

| 有多个未完成 | 展示最近1个 + "查看全部 >"链接 → 我的作品 |
| 首次进入（onboarding后） | 根据记录的 `user_level` / `intent` 调整推荐内容 |

| 提示文案 | 辅助文本 | 底部提示 |

**交互逻辑：**

| 操作 | 行为 |
|------|------|
| 双指缩放 | 调整图片在裁剪框内的范围 |

| 图片格式不支持 | Toast 提示"暂不支持该格式，请选择 JPG/PNG 图片" |

| 按颜色 | 每种颜色为一个步骤，按用量从多到少排列 |
| 按区域 | 图稿等分为 N 个区块（例如 2×2 = 4块），逐块完成 |
| 按行列 | 逐行从上到下，每行为一个步骤 |

**状态逻辑：**

| 场景 | 行为 |
|------|------|
| 首次进入 | 弹出引导方式选择 → 确认后进入第一步 |
| 中途退出 | 自动保存：当前步骤序号 + 引导方式 + 已完成步骤列表 |
| 再次进入 | 恢复到上次步骤，Toast "已恢复到上次进度" |
| 完成最后一步 | → 完成确认弹层 (P-013) |
| 切换引导方式 | 弹窗确认"切换后进度将重新计算，是否继续？" |

**退出逻辑：**

点击"退出"时不强行打断，仅显示轻提示：
```
已自动保存当前进度，下次可以继续 ✓
```
然后返回作品详情页。

- 次按钮：文字/描边
- 危险操作（删除）：主按钮用红色

### 4.2 Toast 提示

- 屏幕上方居中
- 自动消失（2秒）
- 不可交互
- 示例："已保存 ✓"、"已恢复到上次进度"

| 状态 | 颜色 | 文字 |
|------|------|------|
| 草稿 | 灰色 | 草稿 |
| 制作中 | 蓝色 | 制作中 |
| 已完成 | 绿色 | 已完成 |

| 等级 | 显示 | 说明 |
|------|------|------|
| 简单 | ★☆☆ | ≤ 8色, ≤ 400颗 |
| 中等 | ★★☆ | 9-18色, 401-1500颗 |
| 困难 | ★★★ | > 18色 或 > 1500颗 |

### 4.5 加载状态

**页面级加载：**
- 居中显示加载动画 + 文案

**局部加载：**
- 骨架屏（列表页）
- 局部 spinner（按钮提交）

### 4.6 空状态模板

```

                      (中途退出自动保存)
```

| 状态 | 可执行操作 |
|------|-----------|
| 草稿 | 编辑图稿、导出、查看材料、开始跟做、删除 |
| 制作中 | 编辑图稿、导出、查看材料、继续跟做、删除 |
| 已完成 | 查看完成记录、添加实拍、导出、删除 |

### 5.2 免费版使用限制状态

```
转图次数：每月 3 次
  → 已用 0-2 次：正常使用
  → 已用 3 次：触发付费引导弹窗

作品数量：最多 5 个
  → 已有 0-4 个：正常使用
  → 已有 5 个：触发"作品库已满"提示

导出：
  → 免费版：带水印
  → 专业版：无水印
```
